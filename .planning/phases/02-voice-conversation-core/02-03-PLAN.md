    Replace the `setImmediate` handler body with a switch on `eventType`:

    **case 'call.initiated':**
    - Extract `call_control_id` and `call_leg_id` from `event.data.payload`
    - Answer the call: `await getTelnyxClient().calls.answer(call_control_id, { client_state: Base64.encode(JSON.stringify({ source: 'openclaw' })) })`
    - Log: `[webhooks] Answering call ${call_control_id}`

    **case 'call.answered':**
    - Extract `call_control_id` and `from` from payload
    - `initCall(call_control_id, from)`
    - Emit greeting TTS: `await getTelnyxClient().calls.speak(call_control_id, { payload: GREETING.en, voice: ELEVENLABS_VOICE_STRING })`
    - Note: Greeting defaults to English. Language updates after first transcription via detectLanguage.
    - Log: `[webhooks] Greeting emitted for ${call_control_id}`

    **case 'call.speak.ended':**
    - Extract `call_control_id`
    - Get call state. If stage is 'greeting', update stage to 'intake'.
    - Log: `[webhooks] Speak ended for ${call_control_id}, stage: ${state?.stage}`

    **case 'call.hangup':**
    - Extract `call_control_id`
    - `endCall(call_control_id)`
    - Log: `[webhooks] Call ended, cleaned up ${call_control_id}`

    **default:**
    - Log: `[webhooks] Unhandled event: ${eventType}`

    Wrap the entire switch body in try/catch. All Telnyx API calls must be awaited inside the setImmediate async IIFE: change `setImmediate(() => {` to `setImmediate(async () => {`.

    **IMPORTANT:** The Telnyx SDK calls use `calls.answer(callControlId, opts)` and `calls.speak(callControlId, opts)` — NOT methods on a call object. The first argument is always the `call_control_id` string.
