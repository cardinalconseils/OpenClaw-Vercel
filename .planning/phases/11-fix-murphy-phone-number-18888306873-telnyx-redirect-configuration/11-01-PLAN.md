**Step 2 — Harden rawBody extraction in webhook-verify.ts:**

Update `webhook-verify.ts` line 33 to handle the case where Vercel's `shouldAddHelpers: true` pre-parses the JSON body into a JavaScript object. Replace the existing rawBody line:

```typescript
const rawBody = req.body instanceof Buffer ? req.body.toString('utf8') : String(req.body ?? '');
```

With this more robust version:

```typescript
const rawBody = req.body instanceof Buffer
  ? req.body.toString('utf8')
  : typeof req.body === 'string'
    ? req.body
    : typeof req.body === 'object' && req.body !== null
      ? JSON.stringify(req.body)
      : String(req.body ?? '');
```

This ensures that if Vercel pre-parses the body into an object, it gets re-serialized to JSON instead of becoming `"[object Object]"`. Note: this is a defense-in-depth measure. The primary fix is the env var correction.
