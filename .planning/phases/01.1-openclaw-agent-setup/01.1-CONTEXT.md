**Gathered:** 2026-03-14

## Phase Boundary

Install and configure the OpenClaw agent framework, define agent persona (Murphy) and system prompt, wire LLM providers (OpenRouter + Anthropic tiered), create tool registry skeleton, and verify the agent responds to a basic prompt via the gateway and to a simulated webhook event.

<decisions>
## Implementation Decisions

### Agent Persona
- Agent name: **Murphy**
- Tone: Friendly professional — warm but competent, not overly casual
- Murphy identifies as male ("he/him" in system prompt)
- AI disclosure in greeting — exact wording is Claude's discretion, but must comply with FCC/CA SB-1001 requirements
- Greeting style: brief, friendly, gets to the point

### LLM Provider & Model
- **Tiered approach**: cheap model via OpenRouter for routine tasks, Anthropic for complex reasoning
- Both providers wired from day one in Phase 1.1
- OpenRouter handles: greeting, basic intent capture, filler speech, simple responses
- Anthropic handles: complex disambiguation, ranking decisions, transfer logic
- Routing logic between models is Claude's discretion — optimize for voice latency and cost
- Specific cheap model selection via OpenRouter is Claude's discretion — prioritize low latency and good function calling

### Tool Registry Design
- Tool granularity, naming, and organization are Claude's discretion
- Registry pattern and structure (static vs dynamic, aligned with dispatch pipeline or independent) are Claude's discretion
- Phase 1.1 scope for tools: skeleton registry + whatever stubs are needed to verify the agent works — real implementations come in later phases

### Verification Strategy
- Two verification paths: text prompt via gateway AND simulated webhook event
- Text prompt proves: LLM wiring + persona + gateway integration
- Webhook simulation proves: full pipeline from Telnyx event to agent response
- Automated vs manual split and verification depth are Claude's discretion

### Claude's Discretion
- Exact AI disclosure wording in Murphy's greeting
- LLM routing logic between cheap and Anthropic models
- Specific OpenRouter model selection (optimize for voice agent latency)
- Tool granularity, naming, and registry pattern
- Verification test structure (automated, manual, or both)
- Verification criteria depth (specific assertions vs basic response check)
- System prompt structure and content beyond persona decisions above

</decisions>
