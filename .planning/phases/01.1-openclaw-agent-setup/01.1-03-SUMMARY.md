# Phase 1.1 Plan 03: OpenClaw Config Generator and Webhook Wiring Summary

**One-liner:** openclaw.json + Murphy persona files generated at startup, and webhook call.initiated events route to orchestrator chat() via setImmediate.

## What Was Built

### Task 1: OpenClaw config generator and workspace writer (TDD)

`src/startup/openclaw-config.ts` exports two functions:
- `writeOpenclawConfig(options?)` — generates `~/.openclaw/openclaw.json` with gateway (port 18789), dual LLM agents (Murphy), auth profiles (openrouter + anthropic), and voice-call plugin config
- `writeWorkspaceFiles(options?)` — generates `~/.openclaw/workspace/SOUL.md` (Murphy full persona from `buildMurphySystemPrompt()`) and `IDENTITY.md`

`bin/write-openclaw-config.sh` — shell wrapper that invokes both via `npx tsx`.

`bin/sandbox-start.sh` — updated to run config writer in Phase 1.5, before the gateway starts.

8 unit tests use temp directories for full isolation.

### Task 2: Webhook handler wired to orchestrator + integration test

`src/api/webhooks.ts` — imports `chat()` from orchestrator. Inside `setImmediate()`, fires `chat([{ role: 'user', content: 'Incoming call from ${from}. Greet the caller.' }], 'greeting')` on `call.initiated` events.

`tests/lib/ai/agent-integration.test.ts` — 3 tests proving the full pipeline:
1. `call.initiated` → 200 + `chat()` invoked with caller number + 'greeting' task
2. `call.answered` → 200, `chat()` NOT called
3. Unknown event type → 200, `chat()` NOT called
