<task type="auto" tdd="true">
  <name>Task 2: Implement OpenClaw device pre-pairing module with CLI entrypoint</name>
  <files>src/startup/pair-device.ts, tests/startup/pair-device.test.ts</files>
  <behavior>
    - Test 1: prePairDevice() creates the devices directory if it does not exist
    - Test 2: prePairDevice() writes a paired.json file containing an array with at least one entry that has deviceId, publicKey, role="operator", and scopes including "operator.admin"
    - Test 3: prePairDevice() clears pending.json to an empty array
    - Test 4: prePairDevice() writes a device-key.pem file containing a PEM-encoded private key
    - Test 5: Running prePairDevice() twice appends a second entry to paired.json (does not overwrite)
    - Test 6: deviceId is the SHA-256 hex digest of the public key DER bytes
  </behavior>
  <action>
    Create `src/startup/pair-device.ts` implementing the `prePairDevice()` function per the research pattern (Pattern 1 from 01-RESEARCH.md):

    1. Determine `OPENCLAW_STATE_DIR` from env var or default to `~/.openclaw`. Determine `DEVICES_DIR` as `{STATE_DIR}/devices`.
    2. `mkdirSync(DEVICES_DIR, { recursive: true })`.
    3. Generate Ed25519 keypair using `node:crypto.generateKeyPairSync('ed25519', ...)`. Public key in DER/SPKI format, private key in PEM/PKCS8 format.
    4. Compute `deviceId` as `SHA-256(publicKeyDER).hex`.
    5. Build paired entry object: `{ deviceId, publicKey: base64(publicKeyDER), displayName: 'openclaw-sandbox-device', role: 'operator', scopes: ['operator.admin', 'operator.approvals', 'operator.pairing', 'operator.write', 'operator.read'], approvedAtMs: Date.now() }`.
       - Include `operator.write` and `operator.read` scopes per OpenClaw issue #23006 (required for tool connections).
    6. Read existing `paired.json` or default to `[]`. Append new entry. Write back.
    7. Write `[]` to `pending.json` (clear stale requests).
    8. Write private key PEM to `{STATE_DIR}/device-key.pem`.
    9. Return `{ deviceId, publicKey }` for logging.
    10. **CLI entrypoint:** Add a block at the bottom of the file that detects direct execution and runs `prePairDevice()`. Use `import.meta.url` comparison with `process.argv[1]` (via `node:url.fileURLToPath`) to detect if the module is being run directly. When run directly, call `prePairDevice()`, log the returned `{ deviceId, publicKey }`, and exit. This is required because `bin/sandbox-start.sh` (Plan 01-03) invokes this file as `npx tsx src/startup/pair-device.ts`.

    ```typescript
    // CLI entrypoint — allows running as: npx tsx src/startup/pair-device.ts
    import { fileURLToPath } from 'node:url';
    if (process.argv[1] === fileURLToPath(import.meta.url)) {
      const result = prePairDevice();
      console.log(`[pair-device] Paired device ${result.deviceId}`);
    }
    ```

    For tests: Use a temp directory (via `node:os.tmpdir()` + `node:fs.mkdtempSync`) as `OPENCLAW_STATE_DIR` so tests don't touch real filesystem. Set the env var before calling `prePairDevice()`, restore after.
  </action>

  </verify>
