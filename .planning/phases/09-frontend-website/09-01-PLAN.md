  <action>
Replace vercel.json with the builds + routes structure for Express + Next.js coexistence:

```json
{
  "version": 2,
  "builds": [
    {
      "src": "src/server.ts",
      "use": "@vercel/node"
    },
    {
      "src": "frontend/package.json",
      "use": "@vercel/next"
    }
  ],
  "routes": [
    { "src": "/webhooks/(.*)", "dest": "src/server.ts" },
    { "src": "/health", "dest": "src/server.ts" },
    { "src": "/api/(.*)", "dest": "src/server.ts" },
    { "src": "/(.*)", "dest": "frontend/$1" }
  ]
}
```

Key decisions:
- `/webhooks/*` and `/health` route to Express (existing backend endpoints)
- `/api/*` also routes to Express as a future API surface
- Everything else (`/(.*)`) falls through to Next.js
- The `@vercel/node` builder handles TypeScript compilation for Express
- The `@vercel/next` builder handles the Next.js build in `frontend/`

Note: The root `/` route that currently returns JSON service info from Express will now be handled by Next.js (the landing page). The Express root handler remains in server.ts but will only be hit via direct `/api/` prefix if needed.
