  <behavior>
    - MissionScheduler.enqueue(missionId, steps) adds steps to internal queue and starts processing
    - Steps are processed sequentially (not in parallel)
    - Before executing a 'sms' step, scheduler calls smsLimiter.acquire()
    - Before executing a 'call' step, scheduler calls callLimiter.acquire()
    - Before executing a 'search' step, no rate limiter is needed
    - Each step execution calls executeTool with appropriate tool name and params
    - Step status updates to 'in-progress' before execution and 'completed' or 'failed' after
    - Step results are persisted via updateMissionEvent
    - onStepComplete callback is invoked after each step with step id and result
    - onMissionComplete callback is invoked when all steps for a mission finish
    - If a step fails, it is marked 'failed' and processing continues to next step (no abort)
    - isProcessing() returns true while queue is being processed
  </behavior>

       - `export class MissionScheduler`:
         - `private queue: Array<{ missionId: string; step: MissionStep }> = [];`
         - `private processing = false;`
         - `onStepComplete?: (stepId: string, result: Record<string, unknown>) => void;`
         - `onMissionComplete?: (missionId: string) => void;`

         - `async enqueue(missionId: string, steps: MissionStep[]): Promise<void>`:
           - Push each step as `{ missionId, step }` to queue
           - If not already processing, call `this.processNext()`

         - `private async processNext(): Promise<void>`:
           - Set `this.processing = true`
           - While queue has items:
             - Shift item from queue
             - Get limiter from STEP_LIMITER_MAP
             - If limiter exists, `await limiter.acquire()`
             - Call `await this.executeStep(item.missionId, item.step)`
           - Set `this.processing = false`
           - Check if all steps for current mission are done (no more in queue with same missionId), call onMissionComplete if so

         - `private async executeStep(missionId: string, step: MissionStep): Promise<void>`:
           - Update step status to 'in-progress' via `updateMissionEvent(step.id, { status: 'in-progress', startedAt: new Date().toISOString() })`
           - Build tool params from step: for 'search' `{ service_type: step.context, location: step.target }`, for 'call' `{ phone_number: step.target, provider_name: step.context }`, for 'sms' `{ to: step.target, message: step.context }`
           - try: `const result = await executeTool(toolName, params)` then update step to 'completed' with result
           - catch: update step to 'failed' with `{ error: err.message }`, log `[missions:scheduler] Step ${step.id} failed: ${err.message}`
           - Call `this.onStepComplete?.(step.id, result || { error: err.message })`

         - `isProcessing(): boolean` — returns `this.processing`
         - `getQueueLength(): number` — returns `this.queue.length`

       - Export singleton: `export const missionScheduler = new MissionScheduler();`
